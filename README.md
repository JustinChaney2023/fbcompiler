## Language Features

### Data Types
- `int` - integers (e.g., `5`, `42`)
- `float` - floating point (e.g., `3.14`, `10.0`). **Note:** requires digits on both sides of decimal (`.5` and `5.` are invalid)
- `bool` - boolean values (`true`, `false`)

### Operators
| Type | Operators |
|------|-----------|
| Arithmetic | `+` `-` `*` `/` `%` |
| Comparison | `==` `<` `>` `<=` `>=` |
| Logical | `!` `&&` `\|\|` |
| Assignment | `=` |

### Control Flow
- `if` / `else` - conditional branching
- `while` - loops

### Built-in Functions
- `sin(x)` - sine function
- `cos(x)` - cosine function  
- `tan(x)` - tangent function
- `print(x)` - output variable to console (adds newline)

### Comments
```
// single-line comments are supported
```

## Building

### Prerequisites
- flex
- gcc

### Build the Lexer (Current)
```bash
# Generate lexer C code
flex lexer.l

# Compile test driver
gcc lex.yy.c main_test.c -o lexer_test

# Run tests
echo "int x = 5;" | ./lexer_test
./lexer_test < myprogram.txt
```

### Build Full Compiler (When Parser Ready)
```bash
# Once Aaron adds parser.y:
bison -d parser.y          # Generate parser
flex lexer.l               # Generate lexer
gcc parser.tab.c lex.yy.c -o compiler
```

## Testing

### Basic Token Test
```bash
echo "int x = 5;" | ./lexer_test
```
Expected output:
```
line  1 | INT_TYPE       | int
line  1 | IDENTIFIER     | x  => x
line  1 | ASSIGN         | =
line  1 | INT_LITERAL    | 5  => 5
line  1 | SEMICOLON      | ;
```

### Test Float Format Validation
```bash
# Valid - accepted
echo "float y = 3.14;" | ./lexer_test

# Invalid - rejected (error on dot)
echo "float a = .5;" | ./lexer_test
echo "float b = 5.;" | ./lexer_test
```

### Test Complex Expression
```bash
echo "z = (x && y > 3.6) || (y < 5.6);" | ./lexer_test
```

### Test Trig Functions
```bash
echo "float w = sin(x) + cos(y);" | ./lexer_test
```

### Test File
Create `test.txt`:
```
int x = 5;
float y = 3.14;
if (x < y) {
    print x;
}
```
Run:
```bash
./lexer_test < test.txt
```

## Tokens

| Token | Description |
|-------|-------------|
| `INT_TYPE` `FLOAT_TYPE` `BOOL_TYPE` | Type keywords |
| `IF` `ELSE` `WHILE` | Control flow |
| `SIN` `COS` `TAN` | Trig functions |
| `PRINT` | Output statement |
| `TRUE` `FALSE` | Boolean literals |
| `IDENTIFIER` | Variable names |
| `INT_LITERAL` `FLOAT_LITERAL` | Numeric values |
| `ASSIGN` `EQ` `LE` `GE` `LT` `GT` | Assignment/comparison |
| `NOT` `AND` `OR` | Logical operators |
| `PLUS` `MINUS` `MULT` `DIV` `MOD` | Arithmetic |
| `LPAREN` `RPAREN` `LBRACE` `RBRACE` `SEMICOLON` | Punctuation |

## Example Programs

### Variable Declaration & Assignment
```
int x = 5;
float y = 1.4;
bool z = true;

x = 6;
y = 4.3;
z = false || y >= 5.0;
```

### Branching
```
if (x > 6 || z) {
    print(a);
}
else {
    print(tan(b));
}
```

### Looping
```
while (x == 5 && y) {
    print(x);
}
```

### Nested Expressions
```
float w = sin(x) + tan(y);
z = (z && y > 3.6) || (y < 5.6) || (x > 5);
```

## Notes

- Whitespace is ignored (including newlines)
- Identifiers must start with letter or underscore, followed by letters/digits/underscores
- Keywords are case-sensitive (must be lowercase)
- Non-zero integers are treated as true in boolean contexts
- Comments start with `//` and go to end of line
